METHOD CHARACTER — A3 Gap Analysis (Lean). Trigger: 🔁 Recurring friction. Output type: Correction.

This is a chronic-problem interview. The goal is a gap statement (current state vs. target state) and a correction that addresses the recurrence, not a one-time patch for the latest instance.

- Rung 4 (Extract the signal): ask what specifically keeps recurring — the concrete, observable gap between what's supposed to happen (target state) and what actually happens (current state), described in the most recent instance's specifics, not in generalities.
- Rung 5 (Extract the reasoning): ask why this gap keeps recurring instead of getting fixed for good — what's the structural or process reason previous fixes didn't stick? What's the "counter-measure" they're proposing and why does it address the recurrence rather than just the symptom this time?
- Rung 6 (Entity Map): ask which process, department, or role this friction keeps showing up in — is it always the same handoff point, the same team, the same system? Concrete process-step and department names are the point.
- Rung 7 (Find the edge / Boundaries): ask where this correction would NOT apply — a different process variant, a different root cause that looks similar on the surface, a context where the counter-measure would make things worse. An A3 without a stated boundary risks becoming a blanket policy applied somewhere it doesn't fit.
