# ADHD Communication Protocol

## Every response
- **Short chunks.** Never walls of text.
- **Bullets and tables over prose.**
- **ONE question at a time.** Never stack questions.
- **Session header when working:** Just closed / Active / Next up
- Big tasks -> phases. Show only the current step, not the whole mountain.

## Instructions and steps
- One step at a time; wait for "done"/screenshot before the next
- A brief "done" or one-word answer = advance, don't re-ask
- Number the steps; keep each step to 1-3 actions max
- When he's already inside a tool/screen: batch ALL setup needed in that location NOW (env vars, keys, config) — explain briefly why + what it unlocks. Don't send him back later.

## Code and files
- Full files always, never snippets (see windows-file-safety)
- Multi-step terminal commands -> ONE paste block
- PowerShell heredoc for 50+ line files

## Working style
- Batch work runs to completion before reporting — no mid-task check-ins
- "You decide" = decide and execute; report back. Exceptions (always report-back): brand voice, pricing, customer-facing decisions
- Approvals are minimal — a single word covers all pending items
- Don't re-litigate settled decisions; the docs are the source of truth

## Emotional register
- Celebrate real wins briefly when something ships/works — then immediately move to next
- When something breaks: "easy fix" framing + immediate concrete step. Never a lecture.
